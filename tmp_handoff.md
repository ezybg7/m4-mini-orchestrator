

## Pantry — production vision-model selection SHIPPED as PR #10 (Claude worker, feat/vision-model-selection)

Task `pantry-vision-model-selection.task`: select + implement a production, cost-effective, no-training vision model for receipt scanning (Gemini Flash paid baseline vs Claude Haiku vs GPT-4o-mini).

**Decision: Claude Haiku 4.5 (`claude-haiku-4-5-20251001`) via Anthropic Messages API forced tool use** as the production vision provider. Gemini kept as a one-env-var fallback; mock stays for dev. Rationale (ranked): (1) **privacy is the decider — Anthropic no-training is the DEFAULT, not a paid-tier setting**, so unlike Gemini there's no "wrong tier" that trains on a user's receipt photo (the exact free-tier state dev has run in); (2) top-tier thermal-receipt OCR (co-leads Gemini, beats GPT-4o-mini); (3) provider consolidation — app already uses `claude-sonnet-5` for other AI; (4) cost competitive (~$0.008/scan, undercuts the `gemini-3.5-flash` the alias resolves to; trivial at the 20/day cap). GPT-4o-mini rejected (weakest hard-receipt vision + 3rd vendor); local VLM rejected (accuracy risk + ops burden) → removed specs/local-vision-model.md.
